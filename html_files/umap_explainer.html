Step 1 — Extract Cα coordinates from CIF

Each CIF file is an mmCIF record from the Protein Data Bank containing all heavy atoms, metadata, and crystallographic information. We discard everything except the alpha-carbon (Cα) atom of each residue — one point per residue in 3D space. Cα atoms are sufficient to capture the protein backbone geometry. 

import gemmi

def get_ca_coords(cif_path):
    st = gemmi.read_structure(cif_path)
    coords = []
    for chain in st[0]:           # first model only
        for residue in chain:
            for atom in residue:
                if atom.name == "CA":
                    coords.append([atom.pos.x, atom.pos.y, atom.pos.z])
    return np.array(coords)   # shape (N_residues, 3)
Why Cα only? Side-chain atoms add noise without improving backbone geometry representation. Using all atoms would also make structures with different conformations artificially distant.

Step 2 — Build the residue contact graph

A graph G = (V, E) is constructed where each node is a residue (its Cα position) and an edge connects two residues if their Cα atoms are within 8 Å. This threshold captures both local secondary structure contacts and non-local tertiary contacts.

Step 3 — Extract graph-level features

Each contact graph is summarised into a 19-dimensional feature vector capturing topology (how connected residues are), geometry (how spread out they are), and spectral properties (the shape of the Laplacian spectrum).

Feature groupFeatures (dims)What it captures
Size nodes, edges, density (3) How many residues, how many contacts, how densely packed
Degree statistics mean, std, max degree (3) Average contact count per residue and spread — reflects compactness
Clustering coefficient global CC (1) How clique-like local neighbourhoods are — high in helices, low in loops
Distance statistics mean, std Cα–Cα dist (2) Overall spatial spread of the structure
Spectral (Laplacian) 10 smallest eigenvalues (10) Global topology fingerprint — captures connectivity patterns independent of node labelling 
def spectral_features(G, k=10):
    L    = nx.normalized_laplacian_matrix(G).toarray()
    vals = np.linalg.eigvalsh(L)   # deterministic, sorted ascending
    return vals[:k]                  # smallest k eigenvalues
Why Laplacian eigenvalues? The spectrum of the normalised graph Laplacian is invariant to node relabelling — two graphs with identical topology but permuted residue numbering get the same fingerprint. The smallest eigenvalues capture global connectivity (how many clusters, how well-connected), while larger ones capture finer local structure.

Step 4 — Normalise

Features vary wildly in scale: node count is ~300–1300, eigenvalues are in [0, 2], distances in Å. A StandardScaler (zero mean, unit variance per feature) ensures no single feature dominates the distance computation purely because of its scale.

Step 5 — UMAP projection

UMAP (Uniform Manifold Approximation and Projection) reduces the 19-dimensional normalised feature vectors to 2D for visualisation. It works by:

  1. Building a high-dimensional graph — for each structure, find its k=15 nearest neighbours in feature space and weight edges by proximity.
  2. Fuzzy simplicial set — convert distances to a fuzzy membership strength, modelling a manifold in high-D.
  3. Low-D layout — find a 2D embedding that preserves the same fuzzy topology, minimising cross-entropy between the two fuzzy graphs.
What UMAP preserves: local neighbourhood structure (nearby points in 2D were nearby in 19D). What it does not preserve: global distances — the gap between the two clusters in 2D is not proportional to their actual distance in feature space. Use the bar chart (nearest-neighbour distances) for quantitative comparisons, not the 2D positions. 
reducer = umap.UMAP(
    n_components = 2,
    metric       = "euclidean",
    n_neighbors  = 15,    # local neighbourhood size
    random_state = 42,
)
embedding = reducer.fit_transform(feats_scaled)  # (386, 19) → (386, 2)

Key findings

2
structural clusters
Monomers (~310 residues, single catalytic domain) and dimers (~1255 residues, biological assembly). The assembly state is the dominant source of structural variation across all PDE10A entries.
27/27
train/val structures present in PDB download
Every ApherisFold training and validation structure is a strict subset of the 359-structure PDB download. Some are exact duplicates (dist = 0.000).
~0
sequence diversity
All 386 structures share the same protein sequence (query hash 95af60c11e31a0e8). Variation is purely structural — different ligands, crystal forms, and conformations.
19
features per structure
Graph topology (size, degree, clustering) + distance statistics + 10 Laplacian eigenvalues. No sequence information is used — purely 3D contact geometry.
